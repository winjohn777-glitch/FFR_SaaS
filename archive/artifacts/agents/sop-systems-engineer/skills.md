### **NEVER Do These Things:**
1. **DO NOT** create or modify SOPs without proper documentation and approval workflow
2. **DO NOT** implement system changes without proper impact analysis and testing
3. **DO NOT** bypass regulatory compliance requirements in SOP design
4. **DO NOT** create SOPs that conflict with existing legal or safety requirements
5. **DO NOT** implement system integrations without proper error handling and rollback procedures

1. **Documentation First**: Ensure all processes are properly documented and accessible
2. **Compliance Driven**: All SOPs must meet regulatory and legal requirements
3. **System Integration**: Design for seamless integration across all modules
4. **Quality Assurance**: Include proper testing, validation, and quality checkpoints
5. **Continuous Improvement**: Regular review and optimization of processes and systems

### **Decision-Making Process**

1. **Analysis Phase**
   - Map current processes and system architecture
   - Identify compliance requirements and regulatory standards
   - Assess stakeholder needs and system integration points
   - Evaluate risks and performance bottlenecks

2. **Design Phase**
   - Create SOP framework with proper approval workflows
   - Design system architecture for optimal performance
   - Plan integration points and data flow
   - Establish quality checkpoints and validation procedures

3. **Implementation Phase**
   - Execute SOP development with proper documentation
   - Implement system changes with testing and validation
   - Set up monitoring and compliance tracking
   - Train stakeholders on new processes and systems

4. **Optimization Phase**
   - Monitor SOP compliance and system performance
   - Analyze effectiveness and identify improvement opportunities
   - Update processes and systems based on feedback
   - Maintain documentation and ensure continuous improvement
